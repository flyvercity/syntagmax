[COM] The calendar module is the primary time-management surface of Personal Scheduler. It handles the creation, storage, display, and editing of time-bound entries called *events*. An event has at minimum a title and a start time; all other fields are optional.

[COM 1] Recurring events follow the RFC 5545 (iCalendar) recurrence rule syntax to ensure interoperability with third-party calendar applications. Event instances are materialized in the local store only when the relevant time window is displayed or queried, to avoid unbounded storage growth.

[REQ] The application **shall** display events in day, week, month, and agenda views. The currently active view **shall** persist across application restarts.

[REQ] The application **shall** allow a user to create a calendar event by specifying at minimum a title and a start date/time. End date/time, location, description, recurrence rule, and category **shall** be optional fields.

[REQ] The application **shall** support recurring events defined by a recurrence rule conforming to RFC 5545, including DAILY, WEEKLY, MONTHLY, and YEARLY frequency values, with optional UNTIL and COUNT termination conditions.

[COM 2] When the user edits a single instance of a recurring event, the application must offer three choices: edit only this instance, edit this and all following instances, or edit all instances. This prevents accidental modification of the entire recurrence series.

[REQ] The application **shall** support import and export of calendar data in iCalendar (.ics) format as defined by RFC 5545. Imported events **shall** be merged into the existing calendar without duplicating events that already exist based on matching UID fields.
