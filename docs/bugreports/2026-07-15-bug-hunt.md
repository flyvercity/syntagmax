## Confirmed Bugs

### BUG-1 — `--no-git` flag is non-functional (Medium) ✅ FIXED

- **File:** `src/syntagmax/main.py` (lines 56–93)
- **Category:** logic
- **Confidence:** 98%
- **Status:** Fixed — the `process()` function now checks `config.params['no_git']` and skips the `populate_revisions` step when the flag is set.

The `--no-git` CLI flag is declared in `cli.py` and stored in the `Params` TypedDict, but was never consulted anywhere in the execution pipeline. The `populate_revisions` step (which runs `git blame` on every artifact) always executed when `impact` was in the plan, regardless of the flag.

**Runtime trigger:** `syntagmax --no-git analyze impact` — git blame no longer runs.

### BUG-2 — `lstrip('```json')` strips character set, not prefix (Medium) ✅ FIXED

- **File:** `src/syntagmax/ai_providers.py` (lines 218–220)
- **Category:** logic
- **Confidence:** 95%
- **Status:** Fixed — replaced `lstrip('```json').rstrip('```')` with `removeprefix('```json').removesuffix('```')` in all three providers (Ollama, Anthropic, Gemini).

The `OllamaProvider` used `content.lstrip('```json')` to strip a markdown code fence prefix. In Python, `str.lstrip(chars)` treats the argument as a **set of characters** to strip individually, not as a substring prefix. The set `{'`', 'j', 's', 'o', 'n'}` means any leading combination of those characters was removed.

**Runtime trigger:** No longer applicable — `removeprefix`/`removesuffix` correctly strip exact substrings.

### BUG-8 — SidecarExtractor passes non-string YAML values to `add_field` (Medium)

- **File:** `src/syntagmax/extractors/sidecar.py` (lines 100–107)
- **Category:** error-handling
- **Confidence:** 85%

The `add_field` method declares `value: str` in its signature, but Python doesn't enforce type annotations at runtime. The sidecar extractor passes raw YAML values (int, bool, float, None) directly from `yaml.safe_load` output. When a metamodel declares a field as `reference` with `multiple=True`, the code executes `',' in value` which raises `TypeError: argument of type 'int' is not iterable` for numeric values.

**Runtime trigger:** Create a sidecar YAML file with `priority: 3` (integer value) where the metamodel declares `priority` as a reference or multiple field.

**Suggested fix:** Wrap values with `str(value)` before passing to `builder.add_field()`.

### BUG-10 — Case-sensitive `atype` key in sidecar YAML (Low)

- **File:** `src/syntagmax/extractors/sidecar.py` (lines 98–100)
- **Category:** data-integrity
- **Confidence:** 65%

`data.pop('atype', self._record.default_atype)` is case-sensitive. If a user writes `Atype: SYS` or `ATYPE: SYS` in their sidecar YAML, the pop won't find the key, the artifact defaults to the record's `default_atype`, and the original key remains in `data` as a regular field.

This is consistent with YAML spec (keys are case-sensitive) but may confuse users.

**Suggested fix:** Either document the exact key requirement, or add case-insensitive lookup for known reserved keys.
